# 论文架构与目录建议（基于当前项目）

## 一、论文总体架构

1. 问题定义层
- 定义多行为序列推荐任务：输入为 `(u,i,b,t)` 交互序列，输出为下一时刻 Top-K 商品。
- 明确两类核心依赖：
  - 空间依赖：用户-商品-行为构成的异构关系图
  - 时间依赖：行为顺序、间隔与兴趣演化

2. 方法层（ST-GNN）
- 图构建：用户-商品交互图 + 商品转移图
- 空间编码：图传播聚合协同信号
- 时间编码：行为序列与时间间隔联合编码
- 多行为融合：行为类型 embedding 与强度建模
- 目标函数：BPR 排序优化

3. 系统层（工程落地）
- 训练与评估流水线：清洗、训练、对比、消融、敏感性
- 在线推荐系统：交互写入、增量更新、历史淘汰策略
- App 演示系统：双工作区、Inspector、用户行为可视化、注册登录

4. 实验层
- 数据集：Tmall/Beibei（论文）+ 当前演示库（工程）
- 基线：BERT4Rec、LightGCN、MBA 等
- 指标：Recall@K、NDCG@K、转化相关指标、Inspector 质量评分
- 分析：消融、参数敏感性、案例分析（460466 专项）
