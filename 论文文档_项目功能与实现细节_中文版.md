# 项目功能与实现细节（中文正式版）

## 一、项目定位与目标
本项目面向电商场景中的多行为序列推荐任务，围绕“时空图神经网络”开展模型设计、系统实现与应用验证。系统目标是：在用户发生点击、收藏、加购、购买等行为后，能够及时更新用户兴趣表示，并输出更准确的商品推荐结果。

## 二、系统总体功能
本系统采用“算法引擎 + 检验工具 + 桌面应用”三层结构，主要提供以下能力：
1. 多行为时空推荐建模：统一建模图结构依赖与时间演化。
2. 在线交互增量更新：用户行为写入后即时更新推荐列表。
3. 双工作区运行模式：
- 演示数据工作区：直接使用项目内置数据与模型演示。
- 自定义数据工作区：用户导入自己的数据库进行训练与推荐。
4. 自动化检验与评分：批量检查推荐更新是否合理，并导出图表与表格。
5. 用户行为可视化：支持查看指定用户的商品行为统计与最近交互时间。
6. 注册登录机制：提供本地账户注册、登录、退出，保证演示完整性。

## 三、核心算法与实现细节

### 3.1 模型主干
- 空间建模：用户-商品交互图与商品转移图联合传播，提取协同关系。
- 时间建模：基于序列编码器与时间间隔编码捕捉兴趣动态。
- 行为语义建模：区分点击、收藏、加购、购买的行为强度与转化关系。
- 排序优化目标：采用成对排序损失优化推荐排序质量。

### 3.2 在线更新机制
系统支持在界面内提交新交互（点击/收藏/加购/购买），并执行：
1. 写入数据库；
2. 按“每用户历史上限”执行淘汰；
3. 同步更新内存状态；
4. 立即刷新推荐结果。

### 3.3 历史记录淘汰策略
- 默认每用户最多保留 300 条历史交互；
- 系统硬上限为 2000 条；
- 按时间倒序保留最新记录，删除更早记录。

### 3.4 自定义数据库接入规范
自定义表必须包含字段：`u, i, b, t`
- `u`：用户编号（整数）
- `i`：商品编号（整数）
- `b`：行为类型（0点击，1收藏，2加购，3购买）
- `t`：时间戳（Unix 秒级）

系统会自动完成字段校验、范围检查与索引优化，降低用户接入门槛。

## 四、自动检验工具（Inspector）
Inspector 提供三类能力：
1. 批量诊断：随机抽样用户，评估推荐更新质量；
2. 专项商品诊断：对指定商品（如 460466）分析推荐变化；
3. 论文导出：自动生成 CSV、PNG、JSON，便于论文插图与表格引用。

常用指标包括：
- 通过率
- 平均质量分
- 名次提升
- 分值变化量

## 五、桌面应用端功能
应用内主要页面：
1. 账户页面：注册、登录、退出；
2. 演示工作区：直接推荐与交互演示；
3. 自定义工作区：导入数据库、训练、推荐、在线更新；
4. 检验页面：批量评分、专项诊断、结果导出；
5. 用户可视化页面：查看用户对各商品的行为次数与最近时间；
6. 数据页面：查看关键文件、样本规模与行为分布；
7. 论文映射页面：展示模型设计与参考文献对齐关系。

## 六、工程完整性与论文价值
本项目形成了从“理论提出—模型实现—实验评估—应用演示—安装分发”的完整闭环，符合本科毕业论文对工作量、可复现性和工程落地性的要求。

## 七、当前限制与后续扩展
1. 当前账户体系为本地单机账户，暂未实现云端多端同步；
2. 随数据规模扩大，可继续引入分层缓存与分表存储；
3. 后续可扩展多模态特征与超长序列检索机制，进一步提升泛化能力。
